## 들어가며

**직장인 10명 중 8명이 Claude를 아직도 '채팅창'으로만 씁니다.**

"요약해줘", "번역해줘", "이 메일 좀 써줘." 매번 새로운 채팅창에 설명하고, 같은 말을 반복하고, 결과물이 매번 조금씩 달라지는 걸 그냥 감수합니다. 어떤 날은 존댓말, 어떤 날은 반말. 어떤 날은 요약이 5줄, 어떤 날은 10줄.

그런데 Claude에는 이걸 해결하는 기능이 있습니다. **Skills(스킬)**입니다. 아직 국내에서 제대로 다룬 글이 거의 없는 기능인데, 비개발자 직장인에게 가장 현실적인 업무 자동화 방법입니다.

이 글에서 Skills가 뭔지, 어떻게 설정하는지, 실무에 어떻게 쓰는지를 실제로 써본 경험을 바탕으로 정리합니다.

## Claude Skills, 이게 뭔가요

> **[이미지 지시]** Claude.ai Skills 탭 첫 화면 — claude.ai 접속 → 왼쪽 사이드바 하단 프로필 아이콘 클릭 → 'Skills' 탭 선택. Skills 목록이 보이는 화면 전체 캡처. 스킬이 몇 개 있는 상태면 더 좋음.

한 줄로 설명하면 이렇습니다. **"내가 원하는 작업 방식을 Claude에게 저장해두는 기능"**입니다.

예를 들어볼까요. 매번 보고서를 쓸 때마다 이렇게 입력하셨을 겁니다.

> "합쇼체로 써줘. 소제목은 번호 없이. 결론은 마지막에. 200자 이내로."

이걸 매번 타이핑합니다. 또 씁니다. 또 씁니다.

Skills를 쓰면 이 설명을 한 번만 저장합니다. 이후엔 "보고서 스킬"을 선택하고 내용만 넣으면 됩니다. **Claude가 저장된 방식대로 자동으로 처리합니다.**

단골 카페에서 "평소대로요"라고 하면 알아서 만들어주는 것처럼, 내 작업 방식을 기억하는 Claude가 됩니다. 코딩을 몰라도 됩니다. Claude.ai에 접속해서 텍스트로 설정하면 끝입니다.

## 왜 지금 주목받는가

> **[이미지 지시]** Anthropic 공식 발표 화면 — anthropic.com/news 에서 Skills 또는 Claude 업데이트 관련 발표 글 화면 캡처. 또는 Claude.ai의 Skills 소개 팝업/안내 화면.

2025년 말부터 AI가 달라지기 시작했습니다. 채팅창에서 질문하고 답 받는 수준을 넘어서, **실제 업무 파일을 열고 편집하는** 단계로 넘어왔습니다.

Anthropic이 Skills를 공식 발표한 것도 이 시기입니다. 엑셀, 파워포인트, 워드 파일을 Claude가 직접 다루는 게 가능해졌고, 반복 업무를 저장해두는 것도 됩니다.

숫자로 보면 이렇습니다.

- 직장인이 하루 평균 반복 업무에 쓰는 시간: **2.4시간** (McKinsey, 2024)
- 이 중 AI로 자동화 가능한 비중: **45%**
- 맞춤형 자동화 도구 사용 시 체감 효율: 직접 사용 기준 **30~60% 단축**

**"AI가 드디어 내 업무 방식을 기억한다"**는 얘기가 직장인들 사이에서 퍼지기 시작한 이유가 여기 있습니다.

## Skills 설정하는 법 (처음 해보는 분용)

> **[이미지 지시]** Skills 새로 만들기 입력 폼 화면 — Claude.ai > Skills 탭 > '+ 새 스킬' 또는 'Create Skill' 버튼 클릭 후 나오는 이름·설명 입력 화면 전체 캡처. 입력란이 비어있는 상태.

처음이라도 10분이면 됩니다. 단계별로 따라해보겠습니다.

### 1단계: Skills 탭 열기

Claude.ai 접속 → 왼쪽 사이드바에서 내 프로필 클릭 → **Skills 탭** 선택

### 2단계: 새 스킬 만들기

"+ 새 스킬" 버튼 클릭 → 스킬 이름 입력 (예: "주간보고서", "고객메일", "회의록요약")

스킬 이름은 나중에 찾기 쉽게 구체적으로 짓는 게 낫습니다. "보고서1"보다 "금요일주간보고서"가 스킬이 5개 이상 쌓였을 때 훨씬 편합니다.

### 3단계: 스킬 내용 작성 (핵심)

> **[이미지 지시]** 스킬 내용 작성 중인 화면 — 입력 폼에 텍스트가 채워진 상태. 아래 예시 텍스트를 직접 입력해서 화면을 채운 뒤 캡처해도 됨.

Claude에게 "이 작업을 어떻게 해줘"라고 설명하는 텍스트를 씁니다. 예를 들어 주간보고서 스킬이라면 이렇게 씁니다.

```
주간 업무 보고서를 작성합니다.

형식:
- 합쇼체(~입니다, ~했습니다) 사용
- 섹션 순서: ① 이번 주 완료 업무 ② 진행 중 업무 ③ 다음 주 계획 ④ 이슈/요청사항
- 각 항목은 글머리 기호(-)로 정리
- 전체 200~300자 이내
- 이슈가 없으면 ④ 항목은 생략

내가 이번 주 한 일 목록을 보내면 위 형식에 맞게 정리해주세요.
```

어렵게 생각하지 않아도 됩니다. **평소 Claude에게 매번 설명하던 내용을 그대로 붙여넣으면 됩니다.**

### 4단계: 저장 후 사용

저장 버튼 클릭 → 이후 새 채팅에서 해당 스킬 선택 → 내용만 붙여넣기

> **[이미지 지시]** 스킬 선택 후 채팅창 화면 — 새 채팅 열고 Skills 선택 버튼(또는 Skills 드롭다운)을 누른 상태. 스킬 목록이 표시된 화면 캡처. 스킬 이름 목록이 보이면 좋음.

처음 설정에 20분을 투자하면, 이후 수백 번 쓰게 됩니다.

## 직장인이 반복 업무를 없애는 3가지 실전 활용법

### 방법 1) 주간 보고서 초안, 30초 만에 뽑기

> **[이미지 지시]** 보고서 스킬 실행 결과 화면 — 주간보고서 스킬 선택 후 업무 목록(메모 형태) 붙여넣기 → 정리된 보고서 결과물 나온 화면. 인풋과 아웃풋이 한 화면에 보이게 캡처. 결과물 텍스트가 형식에 맞게 나오는 것이 핵심.

매주 금요일 팀장에게 올리는 주간 보고서. 항목은 똑같고 내용만 바뀌는데, 매번 지난 주 보고서를 복붙해서 수정하거나 양식을 다시 설명하고 있지 않으신가요.

Skills를 쓰면 프로세스가 이렇게 바뀝니다.

**기존 방식**
지난 주 보고서 열기 → 내용 수정 → 말투 다듬기 → 항목 빠진 거 없나 체크 → 제출 (**평균 40~50분**)

**Skills 사용 후**
이번 주 한 일 메모 복붙 → 보고서 스킬 선택 → 결과 확인 → 제출 (**8~10분**)

직접 써본 결과, 금요일 보고서 작성 시간이 **45분에서 8분으로 줄었습니다.** 항목을 빠뜨리는 실수도 사라졌습니다.

스킬에 저장하면 효과적인 내용들:
- 합쇼체/해요체 등 말투 지정
- 섹션 순서와 이름
- 글자 수 범위
- 특정 항목은 반드시 포함 (예: "이슈가 없으면 이슈 항목 생략")

### 방법 2) 긴 문서 핵심만 뽑는 요약 스킬

> **[이미지 지시]** 요약 스킬 전/후 비교 화면 — 왼쪽 또는 위: 긴 기획서나 회의록 원문(텍스트가 빽빽한 상태). 오른쪽 또는 아래: 스킬 적용 후 3줄+결정사항+다음할일 형식으로 정리된 결과물. 화면 두 장을 나란히 보여주거나 스크롤 캡처.

기획서, 계약서, 회의록, 리포트. 읽어야 할 문서는 쌓이는데 시간은 없습니다.

아무 설정 없이 그냥 "요약해줘"라고 하면 매번 다른 형식으로 나옵니다. 어떤 날은 5줄, 어떤 날은 3줄. 결정 사항을 빠뜨리기도 합니다. **형식이 달라지면 훑는 속도가 느려집니다.**

요약 스킬에 아래 형식을 저장해두면 달라집니다.

```
문서를 읽고 아래 형식으로 정리해주세요.

- 핵심 3줄 요약 (전체 내용 압축)
- 내가 결정해야 할 것 (Action required)
- 다음 할 일 (Next step)
- 놓치면 안 될 포인트 (있으면 추가, 없으면 생략)
```

형식이 고정되면 두 가지가 생깁니다. 훑는 속도가 빨라지고, 중요한 걸 빠뜨리지 않게 됩니다. **형식을 저장하는 것 자체가 의사결정 속도를 올려줍니다.**

### 방법 3) 고객 메시지 톤, 상황별로 자동 맞추기

> **[이미지 지시]** 상황별 스킬 3개 목록 화면 — Skills 탭에서 "신규고객응대", "파트너사메일", "클레임처리" 3개 스킬이 목록에 보이는 화면 캡처. 스킬 이름이 명확히 보이면 됨. 실제 스킬 이름이 달라도 상관없음.

영업, 고객 응대, 제안서 작성. 상황마다 말투가 달라야 하는데, 매번 "이번엔 좀 더 정중하게", "이건 간결하게"를 따로 설명하고 있지 않으신가요.

상황별로 스킬을 만들어두면 됩니다.

**신규 고객 스킬**
```
신규 고객 첫 응대 메시지를 씁니다.
공감 먼저 → 솔루션 안내 → 다음 단계 제안 순서로.
정중하고 따뜻한 어투. 200자 이내.
```

**파트너사 스킬**
```
파트너사 업무 메일을 씁니다.
핵심만, 간결하게, 친근한 어투.
불필요한 서두 없이 바로 본론부터. 150자 이내.
```

**클레임 대응 스킬**
```
고객 불만 대응 메시지를 씁니다.
사과 → 원인 설명 → 해결책 제시 → 감사 표현 순서로.
감정적으로 공감하되, 명확한 다음 조치를 반드시 포함. 합쇼체.
```

상황을 고르고 내용만 넣으면 됩니다. **말투를 고민하는 시간이 사라집니다.** 브랜드 보이스가 일관성을 유지합니다.

## 직접 써봤습니다

> **[이미지 지시]** 실제 사용 중인 Skills 목록 전체 화면 — Skills 탭에서 본인이 만든 스킬 목록이 보이는 화면 캡처. 스킬 이름들이 나열된 상태면 됨. 없으면 생략 가능.

저는 콘텐츠 기획과 강의 운영을 하고 있습니다. 개발은 전혀 모릅니다.

Skills를 처음 설정하는 데 걸린 시간은 20분이었습니다. 보고서 스킬 하나, 원고 피드백 스킬 하나. 이후 2주 동안 체감한 변화는 세 가지입니다.

**Claude에게 같은 설명을 반복하지 않게 됐습니다.** "합쇼체로, 불필요한 수식어 빼고, 소제목은 번호 없이." 이걸 매번 타이핑하던 게 사라졌습니다. 지금은 스킬 선택하고 내용만 붙여넣습니다.

**결과물에 일관성이 생겼습니다.** 전에는 그날 프롬프트를 어떻게 쓰냐에 따라 결과가 달랐습니다. 지금은 항상 내가 원하는 형식 그대로 나옵니다.

**Claude 쓰는 게 덜 피곤해졌습니다.** 매번 설명하는 게 생각보다 에너지 소모였다는 걸 Skills 쓰고 나서 알았습니다. 작업 자체보다 Claude에게 설명하는 데 더 신경 쓰고 있었던 겁니다.

## 쓰기 전에 알아두면 좋은 것

> **[이미지 지시]** Skills와 Projects 탭 비교 화면 — Claude.ai 사이드바에서 Projects 탭과 Skills 탭이 모두 보이는 화면 캡처. 또는 각각 탭 클릭한 화면 두 장을 나란히.

**Skills는 방식을 저장하는 것이지, 정보를 저장하는 게 아닙니다.**

"우리 회사 제품 정보", "고객사 이름", "팀 내부 규칙" 같은 정보를 기억하게 하려면 Projects 기능을 함께 써야 합니다.

| 기능 | 저장하는 것 | 예시 |
|------|------------|------|
| **Skills** | 작업 방식 | "합쇼체로, 200자 이내로, 결론 먼저" |
| **Projects** | 정보·맥락 | "우리 회사 제품명, 고객사 이름, 팀 규칙" |

두 가지를 함께 쓰면 시너지가 납니다. Projects에 회사 정보를 넣고, Skills에 작업 방식을 저장하면 됩니다.

**처음 설정이 잘 될수록 나중이 편합니다.** 대충 만들면 결과도 대충 나옵니다. 첫 설정에 30분을 투자하면 이후 수백 번 쓰게 됩니다.

**3개월에 한 번은 점검하세요.** 업무 방식이 바뀌면 Skills도 업데이트해야 합니다. 쌓인 스킬을 안 쓰게 됐다면 과감히 지우는 것도 방법입니다.

**스킬 이름은 명확하게 짓는 게 낫습니다.** "보고서1"보다 "금요일주간보고서"가 나중에 찾기 편합니다. 스킬이 5개 이상 생기면 이름이 중요해집니다.

## 마무리

Claude를 채팅 도구로만 쓰고 있다면, 아직 절반만 쓰고 있는 겁니다.

Skills는 코딩 없이, AI 지식 없이도 쓸 수 있는 **가장 현실적인 업무 자동화 도구**입니다. 설정 한 번으로 반복 설명이 사라지고, 결과물이 일관성을 갖게 됩니다.

시작은 스킬 하나면 됩니다. 지금 가장 자주 반복하는 업무 하나를 떠올려보세요. 거기서부터 시작하면 됩니다.
